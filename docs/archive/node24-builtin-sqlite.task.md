# Переход на Node 24 и встроенный node:sqlite

## Проблема

Прод использует `better-sqlite3` — нативный модуль с C++-биндингом. При деплое `npm ci` либо
скачивает прекомпилированный бинарник под платформу/версию Node, либо (если подходящего нет)
компилирует его на месте через `node-gyp`, требуя toolchain (`python3`/`make`/`g++`) на сервере.
Начиная с Node 22.5 в рантайм встроен `node:sqlite` с похожим синхронным API — отдельная нативная
зависимость больше не нужна.

## Желаемое поведение

- Хранилище работает на встроенном `node:sqlite` (`DatabaseSync`) вместо `better-sqlite3`.
- Минимальная версия Node поднята до 24 (на 24 `node:sqlite` доступен без экспериментального
  предупреждения в рантайме; на 22.x код рабочий, но с `ExperimentalWarning`).
- `better-sqlite3` и `@types/better-sqlite3` удалены из зависимостей.

## Риски

- `DatabaseSync` не имеет метода `.transaction()`, как `better-sqlite3` — `removeMany` требует
  ручного `BEGIN`/`COMMIT`/`ROLLBACK`.
- `node:sqlite` формально ещё experimental (RC ожидается в Node 25.7) — возможны breaking changes
  API до стабилизации.
- Прод-сервер (SPB) сейчас на Node v22.20 — деплой этой версии кода потребует установки Node 24
  через `nvm` до выката (учтено в `cicd.plan.md`, этап 2.1).
